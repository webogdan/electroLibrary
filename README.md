# 📚 Electronic Library (C++)

Консольное приложение «Электронная библиотека» на C++, демонстрирующее основные принципы ООП: наследование, инкапсуляцию, абстракцию и полиморфизм.

## Возможности

- Добавление и удаление книг (`Book`) и журналов (`Journal`) в единой библиотеке
- Полиморфный вывод информации о любом элементе через `operator<<`
- Выдача (`borrowItem`) и возврат (`returnItem`) предметов конкретному пользователю
- Отслеживание, какие предметы находятся на руках у пользователя
- Сохранение и загрузка состояния библиотеки в текстовый файл

## Структура классов

```
Item (абстрактный класс)
 ├─ Book     — год, название, доступность, автор, жанр
 └─ Journal  — год, название, доступность, номер выпуска, издатель

User          — id, имя, список названий предметов на руках

Library       — хранит vector<unique_ptr<Item>>, управляет всеми операциями
```

### Принципы ООП в проекте

| Принцип       | Где реализован |
|---------------|-----------------|
| Инкапсуляция  | Все поля классов `private`/`protected`, доступ только через геттеры/сеттеры |
| Наследование  | `Book` и `Journal` наследуются от `Item` |
| Абстракция    | `Item` — абстрактный класс с чисто виртуальными методами `print()`, `getType()`, `serialize()` |
| Полиморфизм   | `Library` хранит `Book` и `Journal` в едином `vector<unique_ptr<Item>>` и работает с ними через общий интерфейс `Item` |

## Файлы проекта

- `head.h` — объявления классов `Item`, `Book`, `Journal`, `User`, `Library`
- `Library.cpp` — реализация методов и `main()` с демонстрацией работы

## Сборка и запуск

```bash
g++ -std=c++17 -Wall -o app Library.cpp
./app
```

## Формат файла сохранения

Каждый предмет библиотеки сохраняется одной строкой, поля разделены символом `|`:

```
BOOK|год|название|доступность(1/0)|автор|жанр
JOURNAL|год|название|доступность(1/0)|номер_выпуска|издатель
```

Пример:

```
BOOK|1972|Green Mile|0|Stephen King|Thriller
JOURNAL|1990|Iron Man|1|1|Marvel Studios
```
